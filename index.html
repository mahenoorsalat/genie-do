Genio-do
Motivation
  • The idea is to relieve developers of the burden of managing the cluster and offer a REST API to interact, similar to tools like Airflow but without Python-specific constraints.
  • In the future, genie-do will support libraries in various languages, enabling custom local actions similar to GitHub Actions, where the community can build and contribute their own actions.
  • This platform aims to bring the functionality of GitHub Actions + AWS Lambda + Apache Spark together.
  • Genie-do is designed as a distributed executor in Kubernetes, orchestrated with Apache Airflow, and is deployable on any cloud provider or on-premises environment. If you can represent your job as a Docker image, we will run it, retry it, distribute it, and much more.
